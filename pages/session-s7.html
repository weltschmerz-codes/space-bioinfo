6.1 Why Network Analysis?

A list of differentially expressed genes is informative, but it treats genes as independent entities. In reality, genes and proteins function as part of highly interconnected networks. A gene's function is often determined not just by its individual activity but by its relationships with other genes — its network neighbors.

Network analysis applied to spaceflight DEG data allows us to: identify hub genes (the most influential regulators); discover gene modules (clusters of co-regulated genes); prioritize candidate therapeutic targets; reveal indirect effects; and integrate multi-omics data.

6.3 Protein-Protein Interaction Network Analysis

The most common workflow for PPI network analysis of DEGs begins with STRING (string-db.org):

  1. Obtain your DEG list: typically significant DEGs (padj < 0.05, |log2FC| > 1) from DESeq2
  2. Upload gene names to STRING → go to string-db.org → 'Multiple Proteins' → paste gene names → select organism → Search
  3. Set interaction confidence threshold: STRING recommends a minimum score of 0.4 (medium confidence) or 0.7 (high confidence)
  4. Export the network: download as a TSV interaction file (node1, node2, score) and node attribute file
  5. Import into Cytoscape for advanced visualization and analysis

6.3.2 Network Topology: Key Concepts

Topological MeasureDefinition & Biological Significance
Degree (connectivity)Number of direct interaction partners of a node. High-degree nodes are called hubs. Hub genes tend to be essential proteins; their disruption has large phenotypic effects.
Betweenness CentralityNumber of shortest paths between all pairs of nodes that pass through this node. High betweenness nodes are 'bridges', critical regulators or bottlenecks.
Closeness CentralityAverage shortest path distance from a node to all others. High closeness = the node can quickly influence the rest of the network.
Clustering CoefficientFraction of a node's neighbors that are also connected to each other. High clustering = the node is embedded in a tight local community (module).
Eigenvector CentralityA node's influence weighted by the influence of its neighbors. Similar to Google PageRank; being connected to important nodes matters more than just having many connections.

6.4 Visualization with Cytoscape

Cytoscape (Shannon et al., 2003) is the standard tool for biological network visualization and analysis. It is a free, open-source desktop application with a rich plugin ecosystem. Key capabilities: import networks from STRING; map node size to degree, node color to log2FC, edge width to interaction confidence; plugins include NetworkAnalyzer (topological measures), MCODE (module detection), and CytoHubba (hub identification).

🖥️ Cytoscape Workflow for Spaceflight DEG Networks
  1. Get DEG list from DESeq2 results (gene names + log2FC values)
  2. Upload DEG names to STRING → export network TSV
  3. In Cytoscape: File → Import → Network → select STRING TSV
  4. Import node attributes: map gene names to log2FC, padj values
  5. Apply visual style: map node color to log2FC (red = up, blue = down), node size to degree
  6. Run NetworkAnalyzer to compute degree, betweenness, closeness
  7. Sort node table by Degree or Betweenness to identify hub genes
  8. Run MCODE to identify modules; annotate modules using clusterProfiler or g:Profiler

6.5 Weighted Gene Co-expression Network Analysis (WGCNA)

WGCNA (Langfelder & Horvath, 2008) is a systems biology method for finding clusters (modules) of genes with highly correlated expression patterns across samples. Unlike PPI networks (which use curated databases), WGCNA is data-driven — it discovers relationships directly from your expression data.

WGCNA works by: computing pairwise Pearson correlations between all genes; constructing a soft-thresholded adjacency matrix; computing the Topological Overlap Matrix (TOM); clustering genes hierarchically; cutting the dendrogram into modules; summarizing each module by its Module Eigengene (ME); and correlating module eigengenes with external traits to identify biologically relevant modules.

⚠️ WGCNA and Small Spaceflight Sample Sizes

WGCNA was designed for large sample sizes (n=20+). Most individual spaceflight experiments have n=3–6. Applying WGCNA to n<10 datasets produces highly unreliable modules. Solutions: combine multiple datasets (meta-analysis) to increase n; use consensus WGCNA across multiple experiments; or stick to STRING-based PPI networks for small n.

6.6 Case Study: Arabidopsis thaliana in Spaceflight Microgravity

Manian et al. (2021) published network analysis of Arabidopsis thaliana grown aboard the ISS using GeneLab datasets GLDS-7 and GLDS-120. Rather than PPI database approaches, they constructed Gene Regulatory Networks (GRNs) directly from expression data using three algorithms: LASSO (penalized regression for sparse regulatory connections), Pearson Correlation GRN, and the HITS algorithm.

Network analysis identified five key root growth-regulating hub genes in spaceflight microgravity GRNs, including genes involved in cell wall synthesis and remodeling, protein transport, and response to auxin. Notably, the phyD mutant adapted to the spaceflight light environment with the lowest transcriptional cost — demonstrating that network analysis can reveal the structural efficiency of a regulatory response, not just the number of genes involved.

Session Summary

  • Network analysis extends beyond DEG lists by revealing how genes interact in coordinated functional modules
  • PPI networks (from STRING) are best for small spaceflight sample sizes; WGCNA requires n≥15–20
  • Hub genes identified by degree and betweenness centrality are the most functionally significant network nodes
  • Cytoscape enables interactive visualization with biological attribute mapping (color to log2FC, size to degree)
  • The Arabidopsis case study demonstrates that network efficiency metrics reveal biology invisible to DEG lists alone
