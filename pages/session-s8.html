7.1 Why Remote Servers and SSH?

Bioinformatics pipelines routinely process gigabytes to terabytes of sequencing data. A typical RNA-seq dataset may occupy 50–200 GB of raw storage and require many hours of computation. Running such analyses on a personal laptop is impractical; local machines typically lack the RAM, CPU cores, and storage capacity required.

Instead, bioinformaticians connect to dedicated remote servers or high-performance computing (HPC) clusters — machines that may have hundreds of CPU cores, terabytes of RAM, and petabytes of storage, all accessible from any location over the internet.

7.1.1 What is SSH?

SSH (Secure Shell) is a network protocol that creates an encrypted connection between your local computer and a remote server. Think of it as a secure tunnel — all commands you type travel encrypted to the server, and all output travels back encrypted to your screen. The basic SSH command is:

ssh username@server_ip_address
# Example:
ssh jdelacruz@192.168.1.100
💡 SSH Key Authentication

Rather than typing a password every time, SSH supports key-based authentication: you generate a key pair (public + private), place the public key on the server, and your private key authenticates you silently. This is more secure than passwords and required for GitHub connections. Session 8 covers SSH key setup in detail.

7.2 Setting Up VS Code for Remote Development

Visual Studio Code (VS Code) with the Remote - SSH extension transforms remote server access. Instead of typing commands blindly into a terminal, you get a full IDE experience — file explorer, syntax highlighting, integrated terminal, and extensions — all running on the remote server.

  1. Install VS Code from code.visualstudio.com
  2. Open Extensions panel (Ctrl+Shift+X) → search "Remote - SSH" → Install
  3. Press F1 → "Remote-SSH: Connect to Host" → enter username@server_ip
  4. VS Code opens a new window connected to the remote server
  5. Open a terminal: Terminal → New Terminal — this is a shell on the remote server
💡 SSH Config File

Save server connection details in ~/.ssh/config to avoid typing the full address each time:

Host philsa-server
    HostName 192.168.1.100
    User jdelacruz
    IdentityFile ~/.ssh/id_ed25519

Then connect with simply: ssh philsa-server

7.3 Essential Linux Navigation

Linux uses a hierarchical file system starting from the root directory /. Your home directory is typically /home/username, abbreviated as ~. All bioinformatics data and scripts will live somewhere under this structure.

7.3.2 Core Navigation Commands

CommandDescriptionExample
pwdPrint Working Directory — shows where you arepwd → /home/jdelacruz
lsList directory contentsls -lh (detailed, human-readable sizes)
cdChange Directorycd ~/data or cd .. (go up one level)
mkdirMake Directorymkdir -p project/data/raw
treeDisplay directory structure as a treetree -L 2 (2 levels deep)

7.3.3 File Management Commands

CommandDescription
cp source destinationCopy a file. Use cp -r for directories.
mv source destinationMove or rename a file or directory
rm filenameDelete a file permanently (no trash/undo). Use with extreme care.
rm -r directoryDelete a directory and all its contents recursively. Very dangerous.
ln -s target linknameCreate a symbolic link (shortcut) to a file or directory
chmod +x script.shMake a script executable
⚠️ rm Has No Undo

The Linux rm command permanently deletes files — there is no Recycle Bin, no undo. Before running any rm command, especially with -r (recursive), always double-check the path with ls first. A common safe practice: use rm -i (interactive mode) to confirm each deletion.

7.3.4 Viewing File Contents

CommandDescription
cat filename.txtDisplay the entire contents of a text file
less filename.txtView file page by page (press q to exit, space for next page)
head filename.txtShow the first 10 lines
head -n 20 filename.txtShow the first 20 lines
tail filename.txtShow the last 10 lines
tail -f logfile.txtFollow a file in real time (useful for monitoring running jobs)

7.4 Checking Workstation Specifications

Before running computationally intensive pipelines, check the resources available on the server. STAR requires ~32 GB of RAM for human genome indexing, so it's critical to verify resources first.

CommandDescription
lscpuDetailed CPU information: cores, threads, architecture, speed
nprocQuick count of available CPU cores/threads
htopInteractive real-time process viewer (press q to exit)
free -hTotal, used, and available RAM in human-readable format
df -hDisk space usage for all mounted filesystems
du -sh .Total size of the current directory
nvidia-smiNVIDIA GPU model, memory usage, utilization (if GPU present)
