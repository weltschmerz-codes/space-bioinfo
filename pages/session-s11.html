11.2 RSEM Gene Quantification

STAR alignment tells us where each read maps. RSEM answers the harder question: how many reads came from each gene? This is complicated by multi-mapping reads — reads that align equally well to two or more genomic locations. Simple counting tools like featureCounts discard these reads entirely. RSEM instead uses an Expectation-Maximization (EM) algorithm to distribute them probabilistically, recovering more signal — particularly important for genes in paralogous families.

💡 EM Algorithm in RSEM

The EM algorithm alternates between two steps: the E-step (estimate how likely each multi-mapping read came from each transcript, given current expression estimates) and the M-step (update expression estimates based on those assignments). After convergence, the result is a set of expected_count values — fractional counts that sum correctly across multi-mappers. These expected counts are what DESeq2 uses.

11.2.1 Building the RSEM Reference Index

Just as STAR requires a genome index, RSEM requires a transcript-centric reference index built from the same FASTA and GTF files. This is a one-time step taking 30–60 minutes:

rsem-prepare-reference \
  --gtf Mus_musculus.GRCm39.112_and_ERCC92.gtf \
  Mus_musculus.GRCm39.dna.primary_assembly_and_ERCC92.fa \
  rsem_ref/Mus_musculus.GRCm39_and_ERCC92

11.2.2 Running Quantification Per Sample

rsem-calculate-expression \
  --bam \
  --paired-end \
  --strandedness forward \
  --estimate-rspd \
  --append-names \
  --no-bam-output \
  -p {threads} \
  {sample}_Aligned.toTranscriptome.out.bam \
  rsem_ref/Mus_musculus.GRCm39_and_ERCC92 \
  rsem_output/{sample}
⚠️ Strandedness Must Match Library Protocol

The --strandedness forward flag tells RSEM that reads originate from the same strand as the gene. Using the wrong strandedness setting can reduce quantification accuracy by 10–30%. If unsure, run infer_experiment.py from RSeQC on one BAM file before running all samples. The GLDS-525 library is confirmed forward-stranded in its assay table on OSDR.

11.3 Pre-DESeq2 Data Cleaning

11.3.1 Removing rRNA Genes

Ribosomal RNA (rRNA) genes are extraordinarily highly expressed — they can represent 80–90% of total cellular RNA. Even after poly-A selection or rRNA depletion during library preparation, some rRNA contamination reaches the count matrix. Including rRNA genes can severely distort DESeq2 normalization and mask biologically meaningful signals. We filter them out using the list of rRNA gene IDs extracted from the GTF in Session 10.

11.3.2 Merging Technical Replicates

In GLDS-525, samples F1, G1, and V5 were each sequenced twice (labeled _techrep1 and _techrep2). Technical replicates come from the same biological sample sequenced at different times — they contain no additional biological information. The correct approach is to sum the counts from both runs, which is equivalent to combining the read files before alignment. After merging, the count matrix has 29 columns (one per biological sample), not 32.

11.4 Differential Expression with DESeq2 (PyDESeq2)

The pipeline uses PyDESeq2, a Python implementation of the DESeq2 algorithm that produces results nearly identical to the R version. DESeq2's statistical approach — negative binomial modeling with empirical Bayes shrinkage of dispersions and fold changes — is particularly well-suited to the small-n design of most spaceflight experiments.

11.4.1 Contrasts: Three Pairwise Comparisons

GLDS-525 has three experimental groups: Space Flight (FLT), Ground Control (GC), and Vivarium Control (VIV). DESeq2 tests all three pairwise contrasts:

ContrastBiological QuestionMost Informative For
FLT vs. GCWhat changes with spaceflight compared to a habitat-matched ground control?Pure spaceflight effect, controlling for habitat hardware
FLT vs. VIVWhat changes with spaceflight compared to standard vivarium housing?Total effect of the ISS environment vs. Earth baseline
GC vs. VIVWhat changes from vivarium to the same habitat hardware on Earth?Hardware/confinement effects independent of spaceflight

11.5 Visualization

11.5.1 Volcano Plots

A volcano plot displays every tested gene as a point — x-axis = log₂ fold change, y-axis = −log₁₀(padj). The "volcano" shape emerges because most genes cluster near the center (small change, uncertain significance), while significant DEGs erupt into the upper-left (downregulated) and upper-right (upregulated) quadrants.

💡 Reading Volcano Plots

Genes that are statistically significant but have tiny fold changes (bottom of the spike) often represent real but biologically minor changes — common in large experiments. Genes in the far upper corners, with both large |log₂FC| AND very small padj, are the most confidently and dramatically changed. These are your primary candidates for biological follow-up.

11.5.2 Principal Component Analysis

PCA is the first plot to generate after normalization — it is your "sanity check." Samples from the same experimental group should cluster together; FLT, GC, and VIV should separate from each other. The pipeline uses VST (Variance-Stabilizing Transformation) counts for PCA, which prevents highly expressed genes from dominating the analysis. If one sample clusters with the wrong group, it may indicate mislabeling, poor RNA quality, or an outlier that should be removed.

11.5.3 Heatmaps of Top DEGs

Heatmaps display the top 50 DEGs per contrast as a color grid (rows = genes, columns = samples). Z-score normalization is applied row-wise so that expression patterns are comparable across genes with very different absolute expression levels. Hierarchical clustering of both rows and columns reveals co-regulated gene modules and sample groupings that emerge from the expression data independently of the experimental design.

11.6 Pathway Analysis: ORA & GSEA

11.6.1 Over-Representation Analysis (ORA)

ORA asks: are the DEGs enriched for genes in specific GO categories or KEGG pathways compared to what random chance would predict? It uses a hypergeometric test against a background of all expressed genes. ORA is fast and intuitive but has a key limitation: it only uses the binary DEG/non-DEG classification, ignoring the continuous fold change signal.

11.6.2 Gene Set Enrichment Analysis (GSEA)

GSEA avoids the arbitrary cutoff problem of ORA by using the entire ranked gene list. All ~20,000 expressed genes are ranked by their log₂FC (most upregulated to most downregulated). For each gene set, GSEA walks this ranked list and calculates a running enrichment score — increasing when a pathway gene appears, decreasing otherwise. The maximum of this running score is the enrichment score (ES). GSEA detects coordinated subtle shifts in expression that ORA might miss.

MethodInputQuestionUse When
ORASignificant DEG listAre DEGs enriched in specific pathways?Quick hypothesis-generating exploration
GSEAAll genes ranked by log₂FCDo pathway genes coordinately shift?Detecting subtle but consistent pathway changes

11.7 Validation Against GeneLab Pre-Computed Results

After running the complete pipeline, we validate our results against the official GeneLab DEG tables for GLDS-525. We compute the Pearson correlation between our log₂FC values and GeneLab's for each contrast. A correlation of r > 0.99 confirms full pipeline reproducibility. Small differences arise from numerical implementation differences between PyDESeq2 and R DESeq2, but these are not biologically meaningful — the gene lists will be nearly identical.
